# 漢字オボエ (KanjiOboe)

일본어 학습자의 한자 암기를 돕기 위해 만든 웹 기반 학습 서비스입니다.  
한자 공부, 테스트, 오답 확인, 진척도 추적 기능을 통해 학습을 지속할 수 있도록 설계했습니다.

## 1. 프로젝트 개요
- 프로젝트명: 漢字オボエ
- 유형: 팀 프로젝트
- 목표: 일본어 학습자가 JLPT 기반 한자를 효율적으로 암기하고 복습할 수 있도록 지원
- 기획 배경:
  기존 한자 학습 서비스들을 참고하여,
  - 진척도 확인 기능
  - 테스트 기능
  - 사용자 친화적인 학습 흐름
  을 보완한 서비스를 제작했습니다.

## 2. 주요 기능
### 회원가입 / 로그인
- 회원 정보 저장
- 로그인 검증
- 닉네임 중복 체크
- 가입일 표시
- 기본 진척도 트래킹

### 공부 기능
- JLPT 시험 범위를 바탕으로 한자 데이터 제공
- 한자, 음독/훈독, 뜻, 예문 학습
- 레벨 / 섹터 / 한자 단위로 학습량 관리
- 학습량 및 남은 학습량 추적

### 테스트 기능
- 객관식 테스트 제공
- 정답 / 오답 즉시 피드백
- 오답 선택 시 사용자의 선택과 정답 표시
- 타이머 종료 시 정답 표시
- 정답/오답 기록 저장
- 섹터 단위 학습 완료 시 출석 체크

### 마이페이지
- 진척도, 정답률, 연속 출석일 확인
- 레벨별 누적 공부량 / 남은 공부량 확인

## 3. 기술 스택
- Frontend: JSP, Scriptlet, CSS
- Backend: Java, Servlet
- Database: Oracle
- Data: Java 기반 크롤링

## 4. DB 구조
- 회원 테이블: userID, userPW, email, attendance, nickname, regDate
- 로그 테이블: 학습 기록, 정오답 여부, 학습 시점, JLPT 레벨, 섹터
- 한자 테이블: 한자, 음독/훈독, 뜻, 예문 등

## 5. 담당 역할
- 부팀장
- 공부 기능 구현
- 한자 DB 모델링 설계/관리

## 6. 프로젝트를 통해 배운 점
- 초기 기획의 구체성이 중요하다는 점
- 역할 분담과 커뮤니케이션의 중요성
- Git 브랜치 관리와 코드 통합의 중요성
- DB 설계와 작업 환경 정의의 필요성

## 7. 실행 방법
1. Oracle DB 설정
2. 테이블 생성
3. 서버(Tomcat) 실행
4. 프로젝트 배포 후 접속
